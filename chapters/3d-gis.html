三维GIS与三维重建详解

从倾斜摄影到 NeRF，从点云处理到 3D Gaussian Splatting——全面解析三维地理空间数据的采集、重建、标准与可视化技术体系。

一、倾斜摄影测量（Oblique Photogrammetry）

倾斜摄影测量是当前城市级三维模型生产的主流技术。与传统垂直航空摄影只获取地面正射影像不同，倾斜摄影通过搭载多个不同角度的相机，同时获取地物的顶部和侧面影像信息，从而自动化生成具有真实纹理的三维模型。

倾斜摄影生成的伦敦三维城市模型（图片来源：Wikimedia Commons）

1.1 五镜头相机系统

标准的倾斜摄影系统通常采用五镜头（Five-Camera）配置：

镜头编号方向倾斜角度主要功能
镜头 1正下方（Nadir）0°（垂直）获取地面正射信息，提供高精度空间定位基准
镜头 2前方（Forward）40°-45°获取建筑物前立面纹理
镜头 3后方（Backward）40°-45°获取建筑物后立面纹理
镜头 4左侧（Left）40°-45°获取建筑物左侧立面纹理
镜头 5右侧（Right）40°-45°获取建筑物右侧立面纹理

典型的工业级五镜头系统包括 Leica CityMapper-2（1.5 亿像素混合传感器）、Trimble MX9、Phase One iXM-RS150F 等。国产系统如 赛尔 PowerVision 和 飞马 D2000 也已在大规模城市三维重建中得到广泛应用。

1.2 倾斜摄影处理流程

从原始影像到最终三维模型，倾斜摄影的完整处理流程包括以下五个核心步骤：

步骤一：空中三角测量（Aerial Triangulation，空三）

空三是倾斜摄影数据处理的基础环节，其目标是建立影像像素坐标与地面三维坐标之间的精确对应关系。核心过程包括：

  1. 特征点提取与匹配：在重叠影像间自动提取 SIFT/SURF 等特征点并进行跨影像匹配
  2. 相对定向：利用匹配点计算相邻影像间的相对位姿关系
  3. 绝对定向：利用地面控制点（GCP）或 GNSS/IMU 观测值将模型转换到大地坐标系
  4. 光束法平差（Bundle Adjustment）：以所有连接点和控制点为约束，同时优化所有影像的外方位元素和连接点的三维坐标
共线方程（Collinearity Equation）—— 空三的数学基础：
x - x₀ = -f · (a₁(X-Xs) + b₁(Y-Ys) + c₁(Z-Zs)) / (a₃(X-Xs) + b₃(Y-Ys) + c₃(Z-Zs))
y - y₀ = -f · (a₂(X-Xs) + b₂(Y-Ys) + c₂(Z-Zs)) / (a₃(X-Xs) + b₃(Y-Ys) + c₃(Z-Zs))

其中：(x, y) 为像点坐标；(x₀, y₀) 为主点坐标；f 为焦距
(X, Y, Z) 为地面点坐标；(Xs, Ys, Zs) 为摄站坐标
a_i, b_i, c_i 为旋转矩阵 R 的元素

光束法平差目标函数：
min Σ_i Σ_j ||x_ij - π(R_i, t_i, X_j)||² + λ Σ_k ||X_k - X_k^GCP||²

步骤二：密集匹配（Dense Matching）

在空三完成后，利用密集匹配算法为每个像素计算三维坐标，生成密集点云。常用算法包括 SGM（Semi-Global Matching）和 MVS（Multi-View Stereo）。SGM 通过多方向动态规划优化视差图，在效率和精度之间取得了良好平衡。

步骤三：网格重建（Mesh Reconstruction）

将密集点云转换为三角网格（Triangulated Irregular Network, TIN）表面模型。常用方法包括 Poisson Surface Reconstruction 和 Delaunay 三角剖分。

步骤四：纹理映射（Texture Mapping）

将原始影像的颜色信息投影到网格表面，生成具有真实纹理的三维模型。需要处理遮挡检测、色彩均衡和接缝消除等问题。

步骤五：模型优化与输出

对三维模型进行简化（Mesh Simplification）、LOD 分级和切片（Tiling），输出为 3D Tiles、OSGB 等标准格式。

1.3 主流倾斜摄影处理软件

ContextCapture（Bentley）

工业级倾斜摄影建模软件，前身为 Acute3D Smart3DCapture。支持超大规模数据集（数万张影像）的分布式处理，是城市级三维重建的首选方案。自动化程度高，输出格式丰富（3D Tiles、OSGB、3MX、OBJ 等）。

Agisoft Metashape

高性价比的专业摄影测量软件（前身为 PhotoScan）。支持倾斜摄影、无人机航测和近景摄影测量。提供 Python API 支持自动化批处理。学术界广泛使用，适合中小规模项目。

其他主要工具还包括 Pix4Dmapper（无人机航测领域市占率最高）、DJI Terra（大疆无人机配套方案）、OpenDroneMap（开源方案）等。

二、LiDAR 点云处理

LiDAR（Light Detection and Ranging，激光雷达）通过发射激光脉冲并测量反射回波的时间来精确获取目标的三维坐标，是除摄影测量外最重要的三维空间数据采集技术。LiDAR 能够穿透植被获取地表信息，在地形测量、林业调查、电力线路巡检等领域具有不可替代的优势。

旧金山街道交叉口的 LiDAR 点云数据（Ouster OS1-64，图片来源：Wikimedia Commons）

2.1 点云滤波算法

点云滤波（Filtering）的目标是将原始 LiDAR 点云分离为地面点（Ground Points）和非地面点（Non-ground Points），是后续 DEM 提取和地物分类的前提。

CSF（Cloth Simulation Filter，布料模拟滤波）

CSF（Zhang et al., 2016）是一种直观且高效的地面滤波算法。其核心思想是将点云上下翻转，然后模拟一块布料在重力作用下覆盖到翻转后的点云表面上——布料最终的形态即近似于地面表面。

CSF 算法步骤：
1. 将点云上下翻转（Z 取反）
2. 在翻转后的点云上方放置一块水平布料网格
3. 模拟布料在重力作用下的下落过程（质点-弹簧模型）：
F_gravity = m · g（重力）
F_spring = k · (||x_i - x_j|| - L₀)（弹簧力）
F_damping = -d · v（阻尼力）
4. 当布料质点接触到点云时停止下落
5. 比较每个点云点与最近布料质点的距离，距离小于阈值的为地面点

参数：布料分辨率（grid resolution）、刚度系数（rigidness）、距离阈值

CSF 的优势在于物理意义直观、参数少且易于调节，已被集成到 CloudCompare 和 LAStools 等主流点云处理软件中。

TIN 渐进加密滤波（Progressive TIN Densification）

渐进 TIN 加密（Axelsson, 2000）是另一类经典的地面滤波方法。算法从稀疏的初始种子点构建 TIN 表面，然后逐步将满足角度和距离阈值条件的点加入 TIN，直到没有新点可加入为止。

渐进 TIN 加密判据：
对于候选点 P 和其投影所在的 TIN 三角面：
- 距离准则：d(P, TIN_face) < d_threshold
- 角度准则：α(P, TIN_edge) < α_threshold

满足两个准则的点被接受为地面点并加入 TIN

2.2 点云分类（Classification）

在地面滤波之后，需要对非地面点进行进一步分类，识别出建筑物、植被、电力线、车辆等不同地物类别。现代点云分类方法主要分为：

  • 基于规则的方法：利用几何特征（高度、平面度、法向量分布）和回波特征（回波次数、强度）设定分类规则
  • 基于机器学习的方法：使用 Random Forest、SVM 等方法对提取的特征进行有监督分类
  • 基于深度学习的方法：直接在原始点云上进行分类，代表方法包括 PointNet（Qi et al., 2017）、PointNet++、RandLA-Net（Hu et al., 2020）和 KPConv（Thomas et al., 2019）

2.3 DEM 提取

基于分类后的地面点云，通过内插生成数字高程模型（DEM, Digital Elevation Model）。常用的内插方法包括：反距离加权（IDW）、克里金（Kriging）、自然邻点法（Natural Neighbor）和TIN 线性内插。LiDAR 生成的 DEM 精度通常可达 5-15 cm（垂直精度），远优于卫星遥感 DEM。

Zhang, W., Qi, J., Wan, P., et al. (2016). An easy-to-use airborne LiDAR data filtering method based on cloth simulation. Remote Sensing, 8(6), 501.
Qi, C. R., Su, H., Mo, K., & Guibas, L. J. (2017). PointNet: Deep learning on point sets for 3D classification and segmentation. CVPR 2017.

三、NeRF：神经辐射场

NeRF（Neural Radiance Fields，神经辐射场）是 Mildenhall 等人于 2020 年在 ECCV 上发表的开创性工作，开辟了基于神经网络的三维场景表示与新视角合成（Novel View Synthesis）的新范式。NeRF 的核心思想是用一个 MLP（多层感知机）网络隐式地表示三维场景的辐射场。

3.1 NeRF 的数学框架

NeRF 将三维场景表示为一个连续的 5D 函数 F，将空间位置 (x, y, z) 和观察方向 (θ, φ) 映射为该点的颜色 (R, G, B) 和体积密度 σ：

NeRF 隐式表示：
F: (x, y, z, θ, φ) → (R, G, B, σ)

其中：
(x, y, z) —— 空间位置坐标
(θ, φ) —— 观察方向（极角和方位角）
(R, G, B) —— 该点在给定观察方向下的颜色（与视角相关，可建模高光反射）
σ —— 体积密度（与视角无关，表示该点是否被物质占据）

NeRF 网络架构：
输入 → 位置编码（Positional Encoding） → 8层MLP(256维) → σ + 特征向量
特征向量 + 方向编码 → 1层MLP(128维) → RGB

位置编码（Positional Encoding）：
γ(p) = (sin(2⁰πp), cos(2⁰πp), ..., sin(2^(L-1)πp), cos(2^(L-1)πp))
将低维输入映射到高维空间，使 MLP 能够表示高频细节

3.2 体积渲染（Volume Rendering）

NeRF 使用经典的体积渲染方程将隐式的辐射场渲染为二维图像。对于从相机发出的每条光线 r(t) = o + td（其中 o 为相机光心，d 为光线方向），沿光线对颜色和密度进行积分：

体积渲染方程：
C(r) = ∫_{t_n}^{t_f} T(t) · σ(r(t)) · c(r(t), d) dt

其中累积透射率：
T(t) = exp(-∫_{t_n}^{t} σ(r(s)) ds)

离散化近似（分层采样 N 个点）：
Ĉ(r) = Σ_{i=1}^{N} T_i · (1 - exp(-σ_i · δ_i)) · c_i
T_i = exp(-Σ_{j=1}^{i-1} σ_j · δ_j)
δ_i = t_{i+1} - t_i（采样间隔）

训练目标：L = Σ_r ||Ĉ(r) - C_gt(r)||²
最小化渲染颜色与真实影像之间的 MSE 损失

3.3 NeRF 的局限与改进

原始 NeRF 存在训练慢（单场景需数小时至数天）、渲染慢（每帧需数秒）和静态场景假设等局限。后续工作进行了大量改进：

改进方向代表工作核心创新
加速训练Instant-NGP（NVIDIA, 2022）多分辨率哈希编码（Multi-Resolution Hash Encoding），训练从小时缩短到秒级
加速渲染MobileNeRF（2022）将 NeRF 烘焙为网格 + 纹理，支持移动端实时渲染
大规模场景Block-NeRF（Waymo, 2022）将城市街景分块建模，支持城市级 NeRF 重建
动态场景D-NeRF（2021）引入时间维度的变形场，建模动态场景
少样本pixelNeRF（2021）从极少视角（甚至单张图像）推断 NeRF 表示
无界场景Mip-NeRF 360（2022）处理无界 360° 场景，多尺度反锯齿
Mildenhall, B., Srinivasan, P. P., Tancik, M., et al. (2020). NeRF: Representing scenes as neural radiance fields for view synthesis. ECCV 2020.
Müller, T., Evans, A., Schied, C., & Keller, A. (2022). Instant neural graphics primitives with a multiresolution hash encoding. SIGGRAPH 2022.

四、3D Gaussian Splatting（三维高斯泼溅）

3D Gaussian Splatting（3DGS）由 Kerbl 等人于 2023 年在 SIGGRAPH 上发表，迅速成为三维重建领域的革命性方法。与 NeRF 使用隐式 MLP 表示场景不同，3DGS 使用大量的显式三维高斯椭球来表示场景，并通过可微分的 tile-based 光栅化实现实时渲染。

4.1 显式 vs 隐式表示

NeRF（隐式表示）

表示方式：MLP 网络权重隐式编码场景
渲染方法：光线行进 + 体积渲染
渲染速度：慢（1-30 秒/帧）
训练速度：慢（小时级）
优势：连续表示、内存紧凑
劣势：渲染慢、编辑困难

3DGS（显式表示）

表示方式：数百万个三维高斯椭球
渲染方法：Tile-based 光栅化
渲染速度：实时（100+ FPS，1080p）
训练速度：快（分钟级）
优势：实时渲染、易于编辑
劣势：内存消耗大、几何精度有限

4.2 高斯椭球参数

每个三维高斯椭球由以下参数定义：

3D Gaussian 参数：

1. 位置均值 μ ∈ R³ —— 高斯中心的三维坐标 (x, y, z)

2. 协方差矩阵 Σ ∈ R³ˣ³ —— 定义高斯椭球的形状和朝向
Σ = R · S · Sᵀ · Rᵀ
其中 R 为旋转矩阵（用四元数 q 参数化），S 为缩放对角矩阵

3. 不透明度 α ∈ [0, 1] —— 该高斯的不透明程度

4. 球谐函数系数 SH —— 表示与视角相关的颜色
使用 l=0,1,2,3 阶球谐函数（共 48 个系数，RGB 各 16 个）
c(d) = Σ_{l=0}^{3} Σ_{m=-l}^{l} c_l^m · Y_l^m(d)

三维高斯函数：
G(x) = exp(-½ (x-μ)ᵀ Σ⁻¹ (x-μ))

4.3 Tile-based 可微分光栅化

3DGS 的核心渲染创新是基于瓦片的可微分光栅化（Tile-based Differentiable Rasterization）：

  1. 投影：将三维高斯投影到二维屏幕空间，得到二维高斯分布
  2. 瓦片分配：将屏幕划分为 16x16 像素的瓦片（tile），每个高斯分配到其覆盖的瓦片中
  3. 排序：在每个瓦片内，按深度对高斯进行排序
  4. Alpha 混合渲染：从前到后遍历每个瓦片中的高斯，进行 alpha 混合计算像素颜色
像素颜色的 Alpha 混合：
C = Σ_{i=1}^{N} c_i · α_i · G_i^{2D}(p) · Π_{j=1}^{i-1}(1 - α_j · G_j^{2D}(p))

其中 G_i^{2D}(p) 为第 i 个高斯在像素 p 处的二维投影值
α_i 为该高斯的不透明度
c_i 为该高斯在当前视角下的球谐颜色

关键优势：整个过程可微分（differentiable），因此可以通过
反向传播梯度来优化所有高斯参数

4.4 自适应密度控制

3DGS 在训练过程中动态调整高斯的数量和分布：

  • 致密化（Densification）：在梯度大的区域（即重建质量不好的区域）克隆（clone）或分裂（split）高斯
  • 剪枝（Pruning）：移除不透明度低于阈值的高斯和体积过大的高斯
  • 不透明度重置：定期将所有高斯的不透明度重置为较低值，让不必要的高斯自然消亡

一个典型的场景最终可能包含 100 万至 500 万个高斯椭球，在 RTX 3090 GPU 上可实现 1080p 分辨率下 100-200 FPS 的实时渲染。

Kerbl, B., Kopanas, G., Leimkühler, T., & Drettakis, G. (2023). 3D Gaussian splatting for real-time radiance field rendering. SIGGRAPH 2023.

五、3D Tiles 与 CesiumJS 生态

5.1 3D Tiles 标准

3D Tiles 是 Cesium 公司提出并被 OGC（Open Geospatial Consortium）采纳的开放标准（OGC Community Standard），专为海量三维地理空间数据的流式传输和可视化而设计。它解决了在 Web 端高效加载和渲染城市级三维模型的核心挑战。

3D Tiles 采用层次化的空间数据结构（通常是八叉树或 KD-Tree），支持基于视点的 LOD（Level of Detail）和视锥体裁剪，实现按需加载。

3D Tiles 内容格式

格式全称用途数据类型
B3DMBatched 3D Model大规模批次化三维模型（如城市建筑群）glTF 几何 + 批次表属性
I3DMInstanced 3D Model实例化模型（如路灯、树木等重复对象）glTF 模板 + 实例位姿矩阵
PNTSPoint Cloud大规模点云数据点坐标 + 颜色/强度/法向量
CMPTComposite复合瓦片，包含多种格式B3DM + I3DM + PNTS 的组合
glTF 2.0GL Transmission Format3D Tiles 1.1 的默认内容格式标准 glTF/GLB + 3DTILES 扩展

glTF 2.0（GL Transmission Format）是 Khronos Group 制定的三维模型传输标准，被称为"三维界的 JPEG"。3D Tiles 1.1 版本将 glTF 2.0 作为默认的内容格式，通过 EXT_structural_metadata 和 EXT_mesh_features 扩展支持地理空间属性信息。这使得 3D Tiles 能够直接利用 glTF 生态系统的工具链（如 Blender、three.js、GPU 压缩纹理等）。

5.2 CesiumJS 生态系统

CesiumJS 是一个开源的 JavaScript 库，用于在 Web 浏览器中创建三维地球和地图。它是 3D Tiles 标准的参考实现，也是目前最成熟的 Web 三维地理空间可视化平台。

CesiumJS 核心能力

全球高精度三维地球渲染；3D Tiles 流式加载与 LOD 管理；支持 WMS/WMTS/TMS 影像服务；Terrain 地形渲染（Quantized Mesh）；CZML 动态时间序列可视化；Entity API 与 Primitive API 双层接口。

Cesium ion 云平台

Cesium 公司的云端三维数据托管与处理服务。自动将倾斜摄影模型、点云、BIM 模型转换为 3D Tiles 格式；提供全球 Terrain 和影像底图服务；支持数据共享和协作；提供 REST API。

CesiumJS 的生态系统还包括：cesium-native（C++ 原生 SDK，用于 Unreal Engine 和 Unity 集成）、Resium（React 封装）、vue-cesium（Vue 封装）等。在中国，SuperMap iClient3D for Cesium 和 Mars3D 是基于 CesiumJS 的主流二次开发平台。

六、CityGML LOD 详解

CityGML 是 OGC 制定的三维城市模型开放标准，基于 GML（Geography Markup Language）扩展。CityGML 定义了 LOD0-LOD4 五个细节层次（Level of Detail），系统性地规范了三维城市模型从二维到室内的完整精度体系。

LOD名称几何表达语义内容精度应用场景
LOD0区域模型2.5D 地形表面 + 建筑物足迹（2D 多边形）建筑足迹、用途> 5m区域/国家级概览、GIS 分析
LOD1块体模型建筑物以平顶块体（Box）表示，无屋顶形态建筑高度、楼层数约 5m城市天际线、噪声模拟、日照粗略分析

CityGML 3.0 的重要更新

  • Space 概念引入：用 AbstractSpace（OccupiedSpace / UnoccupiedSpace）统一描述三维空间，替代了 2.0 版本中不同专题模块各自定义空间的方式
  • LOD 体系简化：新的 LOD 体系与几何类型解耦，同一 LOD 下可以使用不同精度的几何表达
  • 点云支持：原生支持点云（PointCloud）作为建筑模型的一种几何表达
  • 动态数据扩展：Dynamizer 模块支持传感器时序数据的集成，为数字孪生提供标准化接口
  • 版本化：支持城市模型的版本管理和变更追踪

七、BIM+GIS 融合应用

BIM（Building Information Modeling）与 GIS 的融合是三维 GIS 从宏观走向微观、从室外延伸到室内的关键路径。两者的融合不仅是数据格式的转换，更是不同空间尺度、不同语义体系和不同业务流程的深度集成。

7.1 融合层次

融合层次描述技术实现典型应用
可视化融合BIM 模型作为三维对象叠加在 GIS 场景中显示IFC → glTF/3D Tiles 转换，CesiumJS 加载城市三维总览、项目位置展示
数据融合BIM 的属性信息与 GIS 的空间属性关联查询IFC → CityGML 语义映射、空间数据库联合查询建筑信息查询、合规性检查
分析融合利用 BIM 的精细信息增强 GIS 的空间分析能力BIM 模型参与 GIS 空间分析（日照、噪声、能耗）城市微气候模拟、建筑能耗分析
流程融合BIM 和 GIS 在项目全生命周期中的协同工作流CDE（Common Data Environment）、openBIM 标准智慧建造、智慧运维

7.2 典型融合应用场景

智慧园区管理

将园区建筑的 BIM 模型与 GIS 地理底图融合，实现从园区全景到建筑室内的无缝浏览。集成 IoT 传感器数据（温湿度、能耗、安防），实现设施智能运维。

地下管网管理

将市政管网的 BIM 模型（包含管径、材质、埋深等信息）嵌入 GIS 的地下空间中。支持管线碰撞检测、断面分析和事故定位。

规划方案评审

将新建项目的 BIM 方案导入城市 CIM 平台，在真实城市环境中进行日照分析、高度限制检查、天际线影响评估和交通影响分析。

应急救援

融合建筑 BIM（室内结构、消防设施布局）与 GIS（周边道路、消防站位置），辅助消防救援的路径规划和力量部署。

学习路径建议

三维 GIS 的学习建议从以下路径入手：(1) 使用 DJI Mini 系列无人机 + Agisoft Metashape 体验完整的倾斜摄影建模流程；(2) 使用 CloudCompare 学习 LiDAR 点云的基础处理操作；(3) 基于 Nerfstudio 框架实践 NeRF 和 3DGS 的训练与渲染；(4) 使用 CesiumJS 搭建三维地球可视化应用，加载 3D Tiles 数据集；(5) 学习 CityGML 标准文档，理解三维城市模型的语义结构。
